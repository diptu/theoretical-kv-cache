## Standing rules

1. **Week 2 is the only gate.** Everything after it is engineering + writing, which you already know you can do. If the math fails twice, simplify the claim — do not spend week 3 stuck on a proof.
2. **Write while you read.** Reading notes go straight into `related.tex`, never into a separate notes file that gets transcribed later.
3. **Pre-register predictions (week 4).** Committing the predicted allocations *before* running the comparison is cheap and makes the result far more convincing.
4. **Scope only shrinks.** Allowed cuts, in order: Qwen model → LongBench subsets → per-head granularity → ablations. Never cut: the lemma, the bound-vs-measured plot, the three-way comparison.
5. **~10 focused hrs/week minimum.** Weeks 5–6 are the heavy compute weeks — queue overnight runs; a 1B model on Colab Pro / a single consumer GPU is sufficient for everything here.
6. **Friday commit ritual.** Every Friday: push, update the README results table, one-line progress note in `notes/log.md`. Eight entries = a public record of consistent research output, which is itself application material.
